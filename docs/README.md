## How to use this folder

1. **Read `01-creative-brief.md` front to back.** This is your north star. Everything in the build refers back to it.
2. **Work through `04-pre-launch-checklist.md`** in parallel with the build — these items (logos, Kristin's headshot, badges) can be gathered while the site is being built.
3. **Follow `05-setup-guide.md`** to get your computer ready before the first build session.
4. **Use `06-next-steps.md`** to know what happens next.
5. **Refer to `02-homepage-structure.md` and `03-copy-and-content-bank.md`** throughout the build whenever specific section details or copy are needed.

## About this document set

This folder is the output of a structured planning conversation covering business positioning, brand identity, visual direction, site architecture, content strategy, and technical setup. It captures 31 decisions across those areas. Everything written here was locked in with explicit confirmation and represents the agreed direction for the site.

If any decision feels wrong in retrospect, almost everything is reversible. Visual decisions (colors, textures, fonts within the palette, button styles) are trivially easy to change. Structural decisions (page count, tech stack) are harder to change but still doable. The only decisions that are effectively permanent are brand-level: name, positioning, and core voice.
